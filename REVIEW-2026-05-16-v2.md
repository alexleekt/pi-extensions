### 2. `allowSkip` missing from `defineTool` schema
**File:** `index.ts:79-162`

The `AskUserParams` interface (used internally) includes `allowSkip?: boolean`, and `Questionnaire.tsx` respects it (`allowSkip = payload.allowSkip ?? false`). However, the `defineTool` parameter schema does NOT include `allowSkip`. This means the LLM has no way to request skip-capable questionnaires ÔÇö the feature only works via internal calls (`/ask-last` sets `allowSkip: true`).

**Impact:** Users can't ask the LLM to create a questionnaire where partial answers are acceptable.

**Fix:** Add `allowSkip` to the `defineTool` parameters:
```ts
allowSkip: Type.Optional(
  Type.Boolean({
    description: "Allow submitting a questionnaire without answering all questions. Default: false",
  }),
),
```

Also document it in the README parameter table.

### 3. `visual-qa.ts` listens for non-existent `"ready"` event
**File:** `scripts/visual-qa.ts:169-171`

```ts
win.on("ready", () => {
  console.log(`  Ôťů ${name} window ready`);
});
```

`types/glimpseui.d.ts` declares only `"message"`, `"closed"`, `"error"` events. There is no `"ready"` event. Depending on the glimpseui implementation, this may silently fail or throw.

**Fix:** Remove the `"ready"` listener. Use `"message"` or `"closed"` for lifecycle tracking instead.

## ­ččí Medium Issues (Should Fix)

### 4. Tailwind CSS build warning
**File:** `package.json:23`

```json
"build:css": "cd webview && npx tailwindcss -i ./src/index.css -o ./src/index.generated.css --content './index.html,./src/**/*.{js,ts,jsx,tsx}'"
```

The `--content` flag provides a comma-separated string, which Tailwind CLI v3 interprets as a single path containing literal commas. This triggers:

```
warn - The `content` option in your Tailwind CSS configuration is missing or empty.
```

While Vite's PostCSS step later generates correct CSS (since `tailwind.config.js` has the proper `content` array), the warning is unprofessional and confusing.

**Fix:** Remove `--content` from the CLI and rely on `tailwind.config.js`:
```json
"build:css": "cd webview && npx tailwindcss -i ./src/index.css -o ./src/index.generated.css"
```

Alternatively, use space-separated paths: `--content './index.html' './src/**/*.{js,ts,jsx,tsx}'`.

### 6. No `sendToGlimpse` helper ÔÇö repeated type assertion
**Files:** All webview components (`SingleSelect.tsx`, `MultiSelect.tsx`, `Freeform.tsx`, `Questionnaire.tsx`)

The pattern `(window as unknown as { glimpse: { send: (data: unknown) => void } }).glimpse.send(...)` appears 14 times. This is error-prone (if the type changes, every occurrence needs updating) and bloats the code.

**Fix:** Extract to `webview/src/util/glimpse.ts`:
```ts
export function sendToGlimpse(data: unknown): void {
  (window as unknown as { glimpse: { send: (data: unknown) => void } }).glimpse.send(data);
}
```

Also consider a typed `sendCancelled()` helper.

### 7. Missing peer dependency `@earendil-works/pi-ai`
**File:** `package.json:51-53`

`index.ts` imports `StringEnum` from `@earendil-works/pi-ai`:
```ts
import { Type, StringEnum } from "@earendil-works/pi-ai";
```

But `package.json` only lists `@earendil-works/pi-coding-agent` in `peerDependencies`. While `pi-coding-agent` transitively depends on `pi-ai`, relying on transitive dependencies is fragile. If `pi-coding-agent` ever drops or changes its `pi-ai` dependency version, this extension breaks.

**Fix:** Add to `peerDependencies`:
```json
"@earendil-works/pi-ai": "*"
```
